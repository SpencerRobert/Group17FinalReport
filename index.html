Executive Summary

The project we chose to work on throughout the course of the semester was a redesign of the Reading Line Chrome Extension.


In terms of user experience, this HCI artifact accomplished most of the goals we covered in the early stages of this course. It provides good utility, it is efficient to use, safe to use, easy to learn and easy to remember. However, the one goal it fails to reach is the goal of being effective to use. In many scenarios this tool is not nearly as effective as it could be. The original developer created a significant constraint for users by failing to incorperate any customization options regarding the reading line. This design flaw is confirmed by the existing user feedback on the chrome store - some of which is pictured below:

In class we learned about questions we needed to ask ourselves when beginning a design project:

Are there problems with an existing product or user experience? If so, what are they?

Yes, we found that problems existed with the original product. In certain scenarios the default specifications caused the reading line to be:

  • a size that is inapplicable to the text on the current webpage
  • a color that reduces effectiveness on the current webpage

and we found that constantly lining up the reading line yourself made for a less enjoyable user experience.


Why do you think there are problems?

We think these problems exist because of the constraint that exists on the reading line - not allowing its properties to be altered by the user.


What evidence do you have to support the existence of these problems?

The evidence we have to support these problems mainly consists of the user feedback that exists on the Chrome Store. We have also tested the original product and experienced the same issues that were described by other users.


How do you think your proposed design ideas might overcome these problems?

We believed our design ideas would overcome these problems because they provided the users exactly what they asked for - which they reccomended would fix their issues.


Once we answered these questions, we had identified the problems and the solutions we needed to implement. We created a storyboard to further illustrate our idea:






With our design phase winding up, we began to develop our prototype. Given the time frame of this project, we felt it wouldn't make sense to spend time creating any sort of low-fidelity prototypes. We dove right into coding and took an evolutionary prototyping approach where we continuously added new features and tested getting closer and closer to the final product. The end result was a high-fidelity prototype that looks and behaves very similar to our intended final product. Feasibility and technical issues resulted in some aspects of the designs being changed however we kept the design principles we learned from class in mind and made good use of them throughout.

System Map

State 1





The first/initial state of the system is where users will start after downloading and unpacking the extension. In this state the user's Chrome browser tab will appear to be unchanged. From here the user can start by navigating to the extensions logo to the right of the search bar and click it to see the Improved Reading Line extension. To easily see the current status of the program (toggled on or off) users can pin the Improved Reading Line icon as seen below:





When the reading line is toggled on, the icon will switch images to one with a red line in it. When toggled off the image will not include the red line.








State 2





The user navigates to state 2 of the system when they toggle the reading line on. This is done by simultaneously pressing ctrl + alt + - and pressing the same key combination in this state will toggle the reading line off and the user will return to the first state. Once in state 2, the default reading line (identical to the one from the original version) will render where the users mouse is on the webpage. From here the user can use the default reading line if they find it suitable for their current webpage and it will follow the user's mouse vertically along it. If the user wants to make adjustments to the reading line (line height, color, transparency, shift amount and width in the future) can navigate to the Improved Reading Line icon and click it to open the adjustment interface, putting the user in state 3 as seen below:





From the second state of the system, the user is also able use the freeze command by simultaneously pressing shift + f, this navigates the user to state 4. The freeze command causes the reading line to freeze in place and it renders a small instruction panel in the top left corner of the web page as seen below:
