Executive Summary

The project we chose to work on throughout the course of the semester was a redesign of the Reading Line Chrome Extension.


In terms of user experience, this HCI artifact accomplished most of the goals we covered in the early stages of this course. It provides good utility, it is efficient to use, safe to use, easy to learn and easy to remember. However, the one goal it fails to reach is the goal of being effective to use. In many scenarios this tool is not nearly as effective as it could be. The original developer created a significant constraint for users by failing to incorperate any customization options regarding the reading line. In class we learned about questions we needed to ask ourselves when beginning a design project:

Are there problems with an existing product or user experience? If so, what are they?

Yes, we found that problems existed with the original product. In certain scenarios the default specifications caused the reading line to be:

  • a size that is inapplicable to the text on the current webpage
  • a color that reduces effectiveness on the current webpage

and we found that constantly lining up the reading line yourself made for a less enjoyable user experience.


Why do you think there are problems?

We think these problems exist because of the constraint that exists on the reading line - not allowing its properties to be altered by the user.


What evidence do you have to support the existence of these problems?

The evidence we have to support these problems mainly consists of the user feedback that exists on the Chrome Store. We have also tested the original product and experienced the same issues that were described by other users.


How do you think your proposed design ideas might overcome these problems?

We believed our design ideas would overcome these problems because they provided the users exactly what they asked for - which they reccomended would fix their issues.
